1. Overview

FlashMaster is an offline-first study application organized around profiles, subjects, topics, subtopics, flash review, quizzes, analytics, and Google Drive sync. The app loads local collections into React state, scopes them by active profile in memory, and persists updates back to IndexedDB through Dexie helper methods.

Core capabilities

  • Profile-based content and progress tracking
  • Flash study using SM-2 style scheduling
  • Quiz sessions with adaptive selection and timers
  • Analytics driven by review history and quiz attempts
  • CSV and ZIP import with preview and validation
  • Google Drive backup / restore
  • Speech synthesis and speech recognition helpers

Current operating assumptions

  • The app uses state-based screen navigation, not route-based navigation.
  • Google login is the default access gate.
  • Question content is loaded from CSV imports and ZIP image imports in the current UI flow.
  • Image questions require explicit media. There is no Google image search, SVG placeholder, or AI fallback.

5. Question and content model

The renderer supports Text, Image, Audio, MCQ, TrueFalse, FillBlank, MultiSelect, Match, Order, Cloze, and Dictation. Question helpers can select answer variants, build alternate prompts, and render media.

Answer model

  • Primary answer plus optional answerVariants
  • Variant headers like answer-capital become labeled answer targets
  • Study mode can rotate which answer label is requested

Media model

  • media can be an image or audio URL, including ZIP-imported data URLs
  • Image questions require explicit media from CSV or ZIP import
  • No search-based, placeholder, or AI image generation remains in the current build
Authoring note: interactive CSV rows can now supply an options column for MCQ, MultiSelect, and Match. ZIP import remains best for filename-driven image decks.

6. Import workflow

7. Sync and backup

Google Drive sync

  • Uses Google OAuth and a Drive folder named FlashMaster
  • Pulls profile files on login
  • Auto-syncs the active profile after a 4 second debounce
  • Manual sync and restore actions exist in Settings

Backup / restore

  • Exports local JSON snapshots from IndexedDB
  • Supports optional AES-GCM encrypted backups
  • Restores by importing JSON back into Dexie and reloading memory state
Sign inAcquire token and user info.
RestoreImport any profile files pulled from Drive.
WorkStudy locally and persist into IndexedDB.
PushUpload the active profile slice to Drive.

8. Configuration

Google OAuth is required for the normal access path. Image import does not require any separate search API keys. 

VITE_GOOGLE_CLIENT_ID=your_google_oauth_client_id

9. Developer workflow

Run locally

npm install
npm run dev

Build and preview

npm run build
npm run preview

Recommended verification checklist

  • Verify Google sign-in.
  • Import a CSV into a subject and confirm topic / subtopic creation.
  • Import a ZIP of images and confirm filenames become answers and folders become subtopics.
  • Run one flash session and one quiz attempt.
  • Export and re-import a local backup.
  • Trigger Drive sync and confirm profile files exist remotely.

10. Maintenance notes

Operational constraints

  • Browser support for IndexedDB and Google auth popups is required.
  • Speech behavior varies by browser because it depends on built-in APIs.
  • Drive sync is entirely client-side.

Code structure notes

  • src/FlashMaster.jsx mixes UI, persistence, algorithms, and integrations.
  • A future refactor would naturally split storage, sync, renderers, and screens.
  • Documentation should be updated when schema versions, import format, or sync payload shape changes.
